## Vision

### 10x Check
Transform from a one-shot report generator into a self-improving trading intelligence system.
The current skill scrapes, scores, reports, and forgets. The 10x version remembers, tracks outcomes,
calibrates signals, and briefs you proactively. The feedback loop (observe -> record -> track -> evaluate -> calibrate)
turns every analysis into training data for the next one.

### Platonic Ideal
Open Claude Code Monday morning, run `/unusual-whales --brief`, and receive:
- Model accuracy: directional prediction accuracy per bucket (NOT portfolio P&L — no trade journaling)
- Signal calibration highlights (which buckets are actually predictive, per-ticker breakdown)
- Active watchlist conditions (VRP windows, regime changes)
- Alerts triggered since last check
A trading copilot that never sleeps, never forgets, and gets smarter every week.

## Key Design Decisions

### Delivery Model (Eng Review Decision)
- **Email (Gmail MCP) = PRIMARY** — rich HTML full report with all analysis sections
- **Discord = SECONDARY** — short summary only (score, recommendation, VRP signal, 1-line summary)
- **Webhook URL REMOVED from repo** — stored only in user config. No fallback to hardcoded.
- If config missing: skip Discord entirely, attempt email only
- Phase 4 produces canonical `AnalysisReport` data structure; email and Discord are thin formatters

### Lazy Outcome Checking (Eng Review Refined)
No cron or daemon. Auto-check runs **in parallel** with the main command (not blocking):
- At invocation start, check DuckDB for analyses older than 5 weekdays without outcome rows
- Process at most **10 pending outcomes** per invocation (cap prevents blocking after long gaps)
- Report results at END of invocation: "Processed 3/3 outcomes"
- `--check` command processes ALL pending (no cap) for manual backlog clearing
- Price source: UW `/api/ticker/{T}/price` → `prev` field (simple JSON, no Playwright needed)
- Trading day heuristic: weekdays only (no holiday calendar — acceptable for single-user tool)
- Idempotency: check "does outcome row exist for this analysis_id?" — if yes, skip

### Config System
- Location: `~/.config/unusual-whales/config.yaml`
- Permissions: `chmod 600` (contains webhook URL)
- Fallback: If config missing, **skip Discord** (NOT fallback to hardcoded URL)
- Migration: `--setup` command creates config. Webhook URL removed from SKILL.md and discord-delivery.md entirely.

### Signal Calibration (Eng Review Refined)
- Minimum **50** analyses with outcomes before showing accuracy data (raised from 20 — avoids overfitting noise)
- Per-**bucket** accuracy (NOT per-signal — avoids double-counting correlated signals)
- Show BOTH aggregate AND per-ticker breakdown (flags thin ticker diversity)
- Caveat text: "Directional accuracy — not causal. Correlated signals within buckets."
- Does NOT auto-adjust weights (human reviews and decides)

### Replay (Eng Review Decision)
- `--replay` renders STORED data only — no re-scoring with current logic
- Scores, trade ideas, reasoning all come from raw_snapshots JSON as-is
- No schema versioning needed (stored data is self-contained)
